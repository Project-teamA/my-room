Evaluation.csの説明(2017年12月27日 更新)

Evaluation.csは風水評価を司るクラスファイルであり，MonoBehaviourを継承している．
* 基本的に，FurnitureManagement.csと同じように空のオブジェクトにアタッチして使用する．(ただしテストを全く行っていないので動作保証は皆無)
* つまりまだ実装の途中であり改良の余地は大いにある

----------------------------------------------------------------------------------------------------------------------------------------------------

アクセス可能な列挙型の紹介

enum Direction { NotPlaced, CanPlaced } 現在の部屋(配置)の方位がどの方位か
 North = 北
 NorthEast = 北東
 East = 東
 SouthWest = 南東
 South = 南
 SouthWest = 南西
 West = 西
 NorthWest = 北西
 Center = 中央

----------------------------------------------------------------------------------------------------------------------------------------------------

アクセス可能なフィールド(メンバ変数)の紹介 (まだアクセス機能を実装していない)

* 各フィールドにアクセスするときはアクセス用のメソッド(関数)があるため, それを利用すること
  例  aaa_というフィールドにアクセスしたいときは○○.aaa_ではなく○○.aaa()と記述する

float health_luck_ = 健康運の値(基準0でプラスになったりマイナスになったり，以降の運気も五行の気もおそらく同じ)
float economic_luck_ = 金運の値
float love_luck_ = 恋愛運の値
float work_luck_ = 仕事運の値
float popular_luck_ = 人気運
 
float all_luck = 総合運(5個の運の合計)

float elements_wood_ = 五行木
float elements_fire_ = 五行火
float elements_earth_ = 五行土
float elements_metal_ = 五行金
float elements_water_ = 五行水

float energy_strength_ = 気の強さ(五行の合計)

float yin_ = 陰
float yang_ = 陽

int house_ID_ = 部屋配置ID(どの部屋配置の家か)
floor_plan_ = 部屋の種類ID(どんな部屋か)

List<FurnitureGrid> furniture_grid_ = 部屋に配置されている家具(配列)
